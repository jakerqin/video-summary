# Video Insight - 产品需求文档 (PRD)

**文档版本：** v1.0
**创建日期：** 2026-02-03
**产品名称：** Video Insight（视频洞察）
**平台：** macOS 桌面应用

---

## 1. 产品概述

### 1.1 用户痛点

现代信息获取方式正在从文字转向短视频，用户主要通过小红书、抖音、B站等平台获取资讯。然而：

- **时间成本高：** 视频量大，浏览耗时，无法快速筛选有价值内容
- **信息留存难：** 看过的内容难以回顾，关键信息容易遗忘
- **学习效率低：** 视频形式不便于做笔记、整理知识点

### 1.2 产品定位

Video Insight 是一款 macOS 桌面应用，通过 AI 技术将视频内容自动转换为结构化的 Markdown 文本摘要，帮助用户：

- 快速浏览视频核心内容
- 建立个人知识库
- 提升信息获取效率

### 1.3 核心价值

- **时间节省：** 10 分钟视频 → 2 分钟阅读摘要
- **内容留存：** Markdown 格式便于管理和检索
- **灵活定制：** 支持多种摘要风格，适应不同学习场景

---

## 2. 目标用户

### 2.1 主要用户画像

**学习型用户：**
- 通过短视频学习知识、技能
- 需要整理学习笔记
- 希望建立个人知识库

**信息筛选型用户：**
- 每天浏览大量视频内容
- 需要快速判断内容价值
- 希望节省时间成本

### 2.2 用户特征

- 拥有 Mac 电脑（16GB+ 内存推荐）
- 熟悉 Markdown 格式
- 使用 Obsidian、Notion 等知识管理工具
- 主要信息来源：小红书、B站、抖音等

## 3. 功能需求

### 3.1 核心功能（MVP）

#### F1: 视频输入
- **本地文件拖拽：** 支持常见视频格式（MP4、MOV、AVI 等）
- **URL 粘贴解析：** 支持小红书视频链接自动下载
- **批量处理：** 支持添加多个视频到处理队列

#### F2: 智能文本提取
- **字幕优先策略：** 自动检测并提取视频内嵌字幕
- **语音识别：** 使用本地 Whisper 模型进行语音转文字
- **多语言支持：** 支持中文、英文等主流语言

#### F3: AI 摘要生成
- **云端大模型：** 使用 MiniMax M2.1 生成高质量摘要
- **预设模板：** 提供 5-10 个内置摘要风格
  - 学习笔记风格
  - 要点提取
  - 详细记录
  - 问答格式
  - 思维导图
- **自定义 Prompt：** 支持用户编写和保存自定义提示词

#### F4: Markdown 导出
- **格式化输出：** 生成结构化 Markdown 文件
- **元数据包含：** 标题、时间、来源链接、处理时间
- **自动保存：** 导出到用户指定目录
- **完成通知：** 处理完成后打开文件所在文件夹

#### F5: 实时进度反馈
- **可视化进度：** 显示当前处理阶段和进度百分比
- **日志输出：** 显示详细处理日志
- **错误提示：** 清晰的错误信息和解决建议

### 3.2 配置功能

#### F6: 设置管理
- **API 配置：** MiniMax API Key 配置
- **输出路径：** 自定义 Markdown 文件保存位置
- **Whisper 模型：** 选择模型大小（base/small/medium）
- **模板管理：** 管理自定义 Prompt 模板

### 3.3 未来扩展（V2+）

- 支持更多视频平台（抖音、B站、YouTube）
- 内置知识库管理（搜索、标签、分类）
- 视频片段标注和引用
- 多语言摘要翻译
- 团队协作和分享功能

## 4. 非功能需求

### 4.1 性能要求

- **处理速度：** 10 分钟视频在 5 分钟内完成处理（含下载）
- **内存占用：** 运行时内存占用 < 4GB
- **启动时间：** 应用启动时间 < 3 秒

### 4.2 兼容性

- **系统要求：** macOS 12.0+
- **硬件要求：**
  - 最低：8GB 内存
  - 推荐：16GB+ 内存
  - 存储：至少 2GB 可用空间

### 4.3 可用性

- **易用性：** 无需技术背景，拖拽即用
- **稳定性：** 单个任务失败不影响其他任务
- **错误恢复：** 支持任务重试和断点续传

### 4.4 安全性

- **隐私保护：** 本地处理，视频不上传云端（仅文本发送到 MiniMax）
- **数据安全：** API Key 本地加密存储
- **网络安全：** HTTPS 通信

---

## 5. 技术约束

### 5.1 技术选型

- **前端：** Electron + React
- **后端：** Python + FastAPI
- **语音识别：** Whisper（本地）
- **文本摘要：** MiniMax M2.1 API
- **视频处理：** FFmpeg

### 5.2 第三方依赖

- **必需：**
  - MiniMax API（付费）
  - FFmpeg（开源）
  - Whisper 模型（开源）

- **可选：**
  - 小红书视频下载器（参考开源项目）

## 6. 成功指标

### 6.1 产品指标

- **用户留存：** 7 日留存率 > 40%
- **使用频率：** 周活跃用户平均处理 10+ 个视频
- **处理成功率：** > 95%

### 6.2 体验指标

- **用户满意度：** NPS > 50
- **摘要质量：** 用户评分 > 4.0/5.0
- **处理速度：** 平均处理时间 < 视频时长的 50%

## 8. 风险与挑战

### 8.1 技术风险

- **平台反爬：** 小红书可能更新反爬策略
  - **缓解：** 参考成熟开源项目，保持更新

- **API 成本：** MiniMax API 调用成本
  - **缓解：** 优先使用字幕，减少 Whisper 使用

### 8.2 产品风险

- **用户习惯：** 用户可能不习惯阅读文本摘要
  - **缓解：** 提供多种摘要风格，满足不同需求

- **竞品压力：** 市场可能出现类似产品
  - **缓解：** 快速迭代，建立用户粘性
